# Recipe: 口播短视频(15–60s,竖屏)—— 旗舰配方

一个"人/角色"对镜头说话的信息型短视频:TTS 配音 + 大字幕 + 动态背景。最常用、最出效果。

## Script rules(Stage 2 的硬标准)
- **钩子公式**:[具体的人/事] + [冲突或反差] + [和观众有关的问题] —— 前 3 秒必须有具体名词,
  禁止"今天我们来聊聊…"式开场。
- **故事+数据**:每 15 秒至少一个具体的名字/数字/事件;抽象概念必须落到例子。
- **句子短**:口语,每句 ≤ 20 字;书面语连词(然而/因此)换成口语(但/所以)。
- **结尾**:一句可带走的结论 + 轻 CTA(关注/评论钩子),不超过 5 秒。
- **TTS 前清洗**:数字转中文读法(2026→二〇二六 或 两千零二十六,按语境)、英文缩写给读法
  (API→A P I 或 接口)、去掉括号旁白。

## Structure(30–60s 版)
| 段 | 时长 | 内容 |
|---|---|---|
| 钩子 | 0–3s | 冲突/反差一句话,标题字大屏 |
| 展开 | 3–15s | 事实 1 + 事实 2(各配一个视觉焦点) |
| 转折/深化 | 15–35s | "但真正的问题是…" 一层反转 |
| 收束 | 最后 5s | 结论一句 + CTA |

## Assembly
1. 起手复制 `templates/koubo-vertical.html`(禁止白板起稿)。
2. TTS → transcribe → 字幕按 `captions.md`;背景用模板的动态层(渐变/粒子/图片 Ken Burns 三选一),
   不要满屏贴素材视频抢字幕焦点。
3. 每段落切换给一个 GSAP 转场(0.3–0.5s),同段内不加花哨动效 —— 动效服务节奏,不是表演。
